# 01-测试基础与方法论

## 知识点

### 测试理论基础
- 软件测试的目的与价值
- 测试分类：黑盒 / 白盒 / 灰盒
- 测试级别：单元 / 集成 / 系统 / 验收
- 测试金字塔（Mike Cohn）与测试冰淇淋反模式
- 测试左移（Shift-Left）与测试右移（Shift-Right）

### 测试方法论
- TDD（测试驱动开发）— Red → Green → Refactor
- BDD（行为驱动开发）— Given / When / Then
- ATDD（验收测试驱动开发）
- 探索性测试（Exploratory Testing）

### 测试策略与计划
- 测试用例设计技术：等价类划分、边界值分析、决策表、状态转换
- 测试覆盖率指标：行覆盖、分支覆盖、条件覆盖
- 风险驱动测试策略
- 测试优先级与ROI评估

### 自动化测试基础
- 什么测试适合自动化？什么不适合？
- 自动化测试的成本与收益分析
- 自动化测试金字塔的实际应用
- 测试脚本的可维护性原则
