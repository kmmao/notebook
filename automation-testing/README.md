## 学习建议

### 推荐学习顺序

1. **先打基础** — 测试理论 + 一门语言的测试框架（推荐 Python + pytest）
2. **Web自动化入门** — Playwright（现代首选）或 Selenium（行业标准）
3. **API测试** — 比UI测试更稳定，ROI更高，优先掌握
4. **CI/CD集成** — 让测试自动跑起来，才是真正的"自动化"
5. **按需深入** — 移动端、性能测试、视觉回归等按项目需要学习

### 核心原则

1. **测试金字塔** — 单元测试 > 集成测试 > E2E测试，越底层越多
2. **不要过度自动化** — 不是所有测试都值得自动化，ROI是关键指标
3. **测试代码也是代码** — 同样需要重构、review、维护
4. **稳定性优先** — 不稳定的自动化测试比没有测试更糟糕
5. **持续学习** — 工具变化快，但测试思维和方法论是不变的
